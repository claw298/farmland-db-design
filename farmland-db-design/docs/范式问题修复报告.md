## 🔧 修复详情

### 问题1: growing_season 表移除冗余字段

**修复前**:
```sql
growing_season (
  id,
  name,
  crop_name,        -- ❌ 冗余字段，违反2NF
  start_time,
  end_time,
  status,
  creat_time
)
```

**修复后**:
```sql
growing_season (
  id,
  name,
  start_time,
  end_time,
  status,
  creat_time
)
```

**理由**:
- `crop_name` 应该通过 `growing_season_crop` 关联表关联到 `crop` 表
- 避免数据冗余和更新异常
- 符合第二范式 (2NF) 要求

**关联表**:
```sql
-- existing table
growing_season_crop (
  id,
  growing_season_id,  -- FK to growing_season.id
  crop_id             -- FK to crop.id
)
```

### 问题2: user 表双ID问题

**修复前**:
```sql
user (
  id,                -- AUTO_INCREMENT 主键
  user_id,           -- ❌ 冗余字段，与 id 功能重复
  user_name,
  phone,
  avatar_url,
  gender,
  name,
  password,
  status,
  created_at
)
```

**修复后**:
```sql
user (
  id,                -- 唯一主键 (AUTO_INCREMENT)
  user_name,         -- 用户名登录
  phone,
  avatar_url,
  gender,
  name,
  password,
  status,
  created_at
)
```

**理由**:
- `id` 和 `user_id` 都是标识符，功能重复
- 保留 `id` 作为数据库主键，使用 `user_name` 作为业务登录名
- 简化表结构，避免混淆

### 问题3: pest_disease_record 缺少关联字段

**修复前**:
```sql
pest_disease_record (
  id,
  name,
  type
  -- ❌ 缺少与 plot 的关联
)
```

**修复后**:
```sql
pest_disease_record (
  id,
  name,
  type,
  plot_id            -- ✅ 添加关联字段
)
```

**理由**:
- 病虫害记录需要关联到具体地块才能使用
- `plot_id` 字段使每条病虫害记录可追溯到具体地块
- 支持按地块查询病虫害历史
