## 🔴 严重问题 (必须修复)

### 1. 【2NF违反】growing_season 表冗余字段

**问题**:
```
growing_season 表包含 crop_name 字段，但 crop_name 应该通过 crop_id 关联到 crop 表
```

**现状**:
```sql
growing_season (
  id, name, crop_name, start_time, end_time, status, creat_time
  -- crop_name 直接存储，违反2NF
)
```

**建议**:
```sql
-- 删除 crop_name 字段
-- 保留 growing_season_crop 关联表
growing_season_crop (
  growing_season_id, crop_id  -- 联合主键
)
```

**影响**: 数据冗余、update异常、删除异常

---

### 2. 【2NF违反】user 表双主键问题

**问题**:
```sql
user (
  id,              -- AUTO_INCREMENT 主键
  user_id,         -- 另一个ID字段
  user_name,
  name,
  password,
  ...
)
```

**分析**:
- `id` 是自增主键
- `user_id` 是另一个标识符（业务主键？）
- 存在部分函数依赖

**建议**:
```sql
-- 方案A: 统一为一个ID
user (
  id BIGINT PRIMARY KEY,  -- 移除 user_id，保留 id 作为唯一标识
  user_name VARCHAR UNIQUE,
  ...
)

-- 方案B: 保留双主键设计（如果业务需要）
user (
  id BIGINT,             -- 业务主键 (UNIQUE)
  user_id VARCHAR,        -- 另一个标识符
  user_name VARCHAR,
  ...
  PRIMARY KEY (id)
)
```

---

### 3. 【1NF违反】pest_disease_record 无关联字段

**问题**:
```sql
pest_disease_record (
  id, name, type
  -- 缺少与 plot/sensor 的关联！
)
```

**建议**:
```sql
-- 方案A: 添加 plot_id
pest_disease_record (
  id, plot_id, name, type, time, ...
)

-- 方案B: 如果是通用病虫害库，保持现状
-- 但需要另一个关联表
pest_disease_report (
  id, plot_id, pest_disease_record_id, time, severity, ...
)
```

---

### 4. 【安全违规】password 明文存储

**问题**:
```sql
user.password 存储为 VARCHAR(255) 明文
```

**建议**:
```sql
user (
  ...
  password_hash VARCHAR(255) NOT NULL,  -- 存储哈希值
  password_salt VARCHAR(64),              -- 盐值
  ...
)
```

**实现**:
- 使用 bcrypt/argon2 哈希
- 不要存储明文密码

---

### 5. 【参照完整性】缺少外键约束

**问题**: 所有外键只是注释标注，未定义为 FOREIGN KEY

**影响**:
- 无法保证参照完整性
- 可能产生孤儿记录
- 级联删除无法自动处理

**建议**:
```sql
-- 示例：添加外键约束
ALTER TABLE weather_daily
ADD CONSTRAINT fk_weather_daily_plot
FOREIGN KEY (plot_id) REFERENCES plot(id)
ON DELETE CASCADE ON UPDATE CASCADE;
```

## ⚠️ 中等问题 (建议修复)

### 6. 【命名不一致】时间字段命名混乱

**现状**:
| 表 | 字段 | 类型 |
|----|------|------|
| base | create_at | VARCHAR |
| plot | created_at | VARCHAR |
| growing_season | creat_time | DATETIME |
| organization | created_at | VARCHAR |
| role | created_at | VARCHAR |

**问题**:
- `create_at` vs `created_at` 不一致
- `creat_time` 拼写错误 (应为 create_time/created_at)
- 使用 VARCHAR 而非 DATETIME

**建议**:
```sql
-- 统一命名和类型
created_at DATETIME DEFAULT CURRENT_TIMESTAMP
updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
```

---

### 7. 【命名不规范】organization.id(key)

**问题**:
```sql
`id(key)`  -- 包含括号，违反命名规范
```

**建议**:
```sql
id BIGINT PRIMARY KEY
```

---
